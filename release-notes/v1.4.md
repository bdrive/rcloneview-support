# RcloneView 1.4 – Release Notes

**Release Date:** March 2026<br/>
**Platforms:** Windows, macOS, Linux

RcloneView 1.4 introduces iCloud Photos integration, a web-based frontend for remote access, enhanced Batch Job capabilities, and numerous stability and usability improvements across all platforms.

## Highlights

- **iCloud Photos Support (Beta, Read-Only):**
  Browse and download photos from your iCloud Photo Library directly within RcloneView.
  View albums, search by date, and download originals — all without leaving the app.
  This beta feature supports read-only access — uploading, deleting, and renaming are not available at this time.

- **Web Frontend (Beta):**
  Access RcloneView from anywhere through your browser. The built-in web server provides a full-featured
  dual-panel file explorer and real-time transfer monitoring.
  Includes password-protected access and an option to allow remote connections.

- **Enhanced Batch Job:**
  Batch Jobs have been significantly upgraded with step-by-step execution, dry run per step,
  variable dependencies between steps, risk level indicators, template search, and full execution history tracking.
  Build complex multi-step workflows with confidence using dry run previews and clear risk visibility.

## New Features

- **Filter Preset Save & Load:**
  Save and load filter presets for reuse across different jobs and sessions.

- **Mount Options on Windows:**
  File and directory permission settings are now available on Windows for mount operations.

## Improvements

- **macOS 12 Support:**
  Restored compatibility with macOS 12 (Monterey).

- **Dropbox Dot-File Exclusion:**
  Option to automatically exclude dot-files (e.g., .DS_Store) when syncing to Dropbox.

- **Linux Tray Icon Compatibility:**
  Improved system tray handling for Linux Cinnamon and other desktops with native tray support.

- **Windows Tray Icon Color:**
  Tray icon color is now adjustable on Windows.

- **Database Path Handling:**
  Improved database path configuration with legacy path support for smoother upgrades.

- **Proton Drive Removed:**
  Proton Drive has been removed from the remote provider list due to upstream changes.

## Bug Fixes

- **Custom Flags Not Applied:**
  Fixed an issue where custom flags were ignored during job execution.

- **Move Job Dry Run:**
  Fixed the Dry Run button not being enabled for Move job types.

- **Terminal Maximized State:**
  Fixed two issues when the terminal panel was maximized — including UI element state errors.

- **Database Initialization:**
  Eliminated duplicate sqfliteFfiInit() calls that could cause intermittent "database closed" errors.

- **Secure Storage Device ID:**
  Fixed device ID handling in secure storage, and resolved a bug where Synology auto-detection disabled state was not saved.

- **Scheduler Input Lag:**
  Fixed UI lag when entering scheduling parameters.
