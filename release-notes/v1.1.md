# RcloneView 1.1 – Release Notes

**Release Date:** November 2025<br/>
**Platforms:** Windows, macOS, Linux

RcloneView 1.1 introduces practical new features and usability enhancements across remote browsing, terminal access, settings management, and interface behavior. This update focuses on improving workflow efficiency and addressing key issues reported by users.

## New Features

- **Built-in Terminal Tab:**
  A brand-new Terminal tab is now available directly inside RcloneView. You no longer need to switch between an external terminal or command prompt — run shell commands, rclone commands, and quick diagnostics right from the app.

- **App Lock (Password Protection):**
  RcloneView can now be protected with an app-level password to prevent unauthorized access.

- **Custom Database Location Support:**
  You can now choose a custom save path for the `rclone_view.db` file, enabling flexible workspace setups, external drive usage, or selective syncing.

- **Remote Manager: Item Size Display in List Mode:**
  List mode in Remote Manager now shows detailed size information for each item.

- **"Copy Full Path (with Remote)" Command:**
  Adds a convenient way to copy the full remote path — including remote name — into the clipboard.

- **Virtual Remote Folder Tree Preview:**
  When adding or editing a virtual remote, the folder tree preview is now displayed to help users pick paths easily.

## UI & UX Improvements

- **Responsive Tab & Window Resizing:** Tabs dynamically resize when the app window is narrowed, preventing clipped or cut-off text.
- **Cloud Folder Browsing Fixes:** iCloud Drive and similar cloud folders now correctly appear in the file browser.
- **Settings Panel Fixes:** Local rclone executable paths no longer disappear unexpectedly.
- **Combine Remote Folder Name Field:** Combine-type remotes now use a dedicated text field for folder name entry for improved clarity.
- **Transfer Window – Dark Mode Fixes:** Resolved color inconsistencies when switching themes.
- **History Panel Improvements:**
  - Error messages are no longer truncated.
  - Scheduled job errors now correctly appear in History.
- **Explorer Layout Enhancements:** Column widths are improved across Explorer views for better readability and alignment.
- **Better Handling of Unsupported Remote Types:** About dialogs for remote types with unsupported APIs now show proper fallback messages.

## Bug Fixes

- **Compare Operation UI Refresh Issue:** Fixed a bug where the UI continued refreshing after Compare-based copy tasks were already completed.
- **Sync/Compare Browser Fix:** Resolved an issue where browsing a remote folder failed when the local disk node was pre-selected.
- **Windows Terminal Path Handling:** Fixed an issue in Windows where rclone configurations were not being read correctly inside the terminal environment.
- **Job Name Validation Error:** Corrected a database exception triggered during job name uniqueness checks.
- **"No Changes" Success Label Fix:** File detail views now correctly display "Success" when no changes were detected during sync/compare.
- **99% Completion Accuracy Rule:** When results are nearly — but not perfectly — identical, the progress indicator now shows 99% instead of incorrectly showing 100%.
