# pCloud White Label Explorer (aperçu privé)

Cette page est un aperçu privé de la manière dont RcloneView peut être proposé sous la forme d'une application de bureau **pCloud Explorer** entièrement personnalisée à la marque de vos utilisateurs.

L'objectif de cette proposition en marque blanche est de :

- Placer la **marque pCloud au premier plan** dans l'ensemble du produit.
- Rendre **la connexion du compte pCloud sans effort pour les utilisateurs** immédiatement après l'installation.
- S'assurer que **pCloud est le premier choix** dans tous les flux de gestion et de navigation.
- Fournir un **support et une maintenance prioritaires** spécifiquement pour les déploiements de marque pCloud.

---

## 1. Kit de marque et intégration visuelle

Nous fournissons un kit de marque où pCloud est la marque principale et visible dans l'ensemble du produit. Toute la marque RcloneView est supprimée ou conservée uniquement lorsque cela est légalement requis (par exemple, les chaînes de version internes).

Éléments clés :

- Nom de l'application défini sur **« pCloud Explorer »** (ou un autre libellé convenu).
- Tous les logos RcloneView remplacés par le **logo pCloud** :
  - Raccourcis de bureau et icônes de la barre des tâches.
  - Icônes de l'installateur.
  - En-tête intégré à l'application et icône de fenêtre.
- Les accents de couleur ajustés pour correspondre à la palette de marque de pCloud, le cas échéant.

## 2. Assistant de connexion pCloud après installation

Immédiatement après l'installation, les utilisateurs sont guidés pour connecter leur compte pCloud afin de pouvoir commencer à utiliser le service sans étapes de configuration supplémentaires.

Comportements principaux :

- À la fin de l'assistant d'installation, le lancement de l'application ouvre **« Ajouter un distant pCloud »** comme flux d'intégration par défaut.
- L'assistant est simplifié et préconfiguré pour pCloud :
  - Le type de fournisseur est présélectionné sur **pCloud**.
  - Seules les options spécifiques à pCloud sont affichées par défaut.
  - Les options avancées restent disponibles derrière un lien **« Afficher les options avancées »**.

<img src="/support/images/en/howto/remote-storage-connection-settings/post-install-pcloud-remote-wizard.png" alt="post install pcloud remote wizard" class="img-large img-center" />

### Assistant d'accès rapide sur l'écran d'accueil

Si l'utilisateur ignore la connexion initiale ou ferme l'assistant :

- Le panneau de droite de l'écran d'accueil affiche une **grande tuile pCloud**.
- Cliquer sur la tuile rouvre l'assistant **« Ajouter un distant pCloud »** à tout moment.

<img src="/support/images/en/howto/remote-storage-connection-settings/on-home-quick-access-wizard.png" alt="on home quick access wizard" class="img-large img-center" />


Cela garantit que la connexion de pCloud reste toujours l'action suivante la plus visible pour les utilisateurs nouveaux ou récurrents.

## 3. Navigation et gestion axées sur pCloud

Une fois qu'un distant pCloud est ajouté, l'interface est optimisée pour garder pCloud bien visible dans tous les écrans clés de navigation et de gestion.

### 3.1 Lecteur pCloud dans RcloneView Explorer

Lorsque le distant pCloud est monté comme lecteur local :

- Le lecteur pCloud (par exemple, `pCloud Drive (P:/)`) est affiché en **haut de la liste des distants locaux de RcloneView**.
- Le lecteur utilise l'**icône pCloud** afin d'être visuellement distinct des autres lecteurs.

<img src="/support/images/en/howto/remote-storage-connection-settings/pcloud-first-in-explorer.png" alt="pcloud first in explorer" class="img-medium img-center" />

### 3.2 Panneau pCloud épinglé à la relance

Une fois qu'un distant pCloud a été configuré :

- Lors des lancements suivants, l'interface s'ouvre avec un **panneau pCloud épinglé** par défaut.
- Disposition typique :
  - Côté gauche : disque local ou une autre source.
  - Côté droit : le distant **MYpCloud** de l'utilisateur.
- Cela rend les opérations répétées de synchronisation ou de copie entre les dossiers locaux et pCloud accessibles en un clic.

<img src="/support/images/en/howto/remote-storage-connection-settings/pcloud-panel-pinned-on-re-launch.png" alt="pcloud panel pinned on re launch" class="img-large img-center" />


### 3.3 pCloud en tête dans « Nouveau distant » et « Gestionnaire des distants »

Pour mettre en avant pCloud comme fournisseur de stockage principal :

- Dans la boîte de dialogue **Nouveau distant** :
  - pCloud apparaît en **haut de la liste des fournisseurs**.
  - La tuile pCloud est visuellement mise en évidence pour encourager sa sélection.
- Dans le **Gestionnaire des distants** :
  - Le distant pCloud (par exemple, `MYpCloud`) est affiché en **haut de la liste des distants**.
  - Dans les vues en liste comme en cartes, le distant pCloud est visuellement mis en valeur pour un accès plus rapide.


<img src="/support/images/en/howto/remote-storage-connection-settings/pcloud-first-in-management-dialog.png" alt="pcloud first in management-dialog" class="img-large img-center" />

## 4. Support et maintenance prioritaires pour pCloud

Pour le déploiement en marque blanche pCloud, nous fournissons un parcours dédié de support et de maintenance.

Services inclus :

- **Documentation dédiée**  
  - Pages de manuel distinctes spécifiquement pour les utilisateurs de **pCloud Explorer**.  
  - Guides pas à pas pour la connexion, la synchronisation et le dépannage avec pCloud.

- **Traitement prioritaire des incidents**  
  - Les problèmes des utilisateurs pCloud sont prioritaires dans la file d'attente du support.  
  - **Réponse d'urgence** pour les incidents critiques affectant les utilisateurs pCloud (par exemple, échecs de connexion, problèmes d'accès aux données).

- **Mises à jour continues du produit**  
  - Mises à niveau régulières du client de bureau incluses dans le cadre de l'accord de maintenance.  
  - Possibilité de déployer de nouvelles fonctionnalités RcloneView sous la marque pCloud après approbation conjointe.

## 5. Prochaines étapes

Si vous souhaitez avancer avec ce produit en marque blanche :

1. **Alignement de la marque**
   - Confirmer les directives d'utilisation du logo pCloud et les couleurs de marque.
   - Décider du nom final du produit (par exemple, *pCloud Explorer*).
2. **Définition de l'expérience**
   - Valider le flux d'intégration et les comportements axés sur pCloud décrits ci-dessus.
   - Ajuster les paramètres par défaut (par exemple, mode de synchronisation par défaut, chemin de montage par défaut).
3. **Version pilote**
   - Nous livrons une version pilote privée et une documentation spécifique à pCloud pour les tests internes.

Cette page et son URL sont destinées uniquement aux parties prenantes de pCloud et à ses partenaires, et n'apparaîtront pas dans la navigation ni la recherche publiques. Elle peut être partagée en toute sécurité sous forme de lien direct lors des discussions d'évaluation et de pilotage.
