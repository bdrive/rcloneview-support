# AWS S3 et compatibles S3

RcloneView prend en charge **Amazon S3** ainsi qu'une large gamme de **fournisseurs de stockage objet compatibles S3**, notamment des acteurs majeurs comme Wasabi, Backblaze, Cloudflare R2, et bien d'autres.

Ces remotes partagent la même interface de configuration basée sur le protocole S3. Une fois sélectionné, vous devrez fournir vos identifiants d'accès et éventuellement une URL de point de terminaison (endpoint) personnalisée.


## Comment ajouter un remote compatible S3

### Étape 1 : Ouvrir la fenêtre de configuration d'un nouveau remote

- Cliquez sur **`+ New Remote`** dans le menu supérieur sous **`Remote`**.
- Vous pouvez aussi cliquer sur le bouton **`+`** dans le panneau Explorer et sélectionner **`New Remote`**.

### Étape 2 : Configurer le fournisseur S3

#### Dans l'onglet **Provider** :
1. Recherchez `s3` ou le nom de votre fournisseur (par exemple `aws`, `wasabi`, `minio`).
2. Sélectionnez **`S3`** dans la liste de résultats.

#### Dans l'onglet **Options** :
3. Définissez votre **provider** (par exemple `AWS`, `Wasabi`, `Minio`, etc.).
4. Saisissez votre [**Access Key ID**](/howto/cloud-storage-setting/aws-account-info#step-by-step-get-your-aws-access-key-id-and-secret-access-key).
5. Saisissez votre [**Secret Access Key**](/howto/cloud-storage-setting/aws-account-info#step-by-step-get-your-aws-access-key-id-and-secret-access-key).
6. Sélectionnez la **region** correcte (par exemple `ap-northeast-2`) pour votre bucket de stockage.
   Si nécessaire, définissez un **endpoint** personnalisé (pour les fournisseurs qui n'utilisent pas le format d'endpoint AWS).

### Étape 3 : Nommer et enregistrer le remote

#### Dans l'onglet **Name** :
7. Attribuez un nom au remote (par exemple `myWasabi`, `minio-prod`).

#### Dans l'onglet **Save** :
8. Cliquez sur **Save** pour finaliser la configuration.

1. Ouvrez le **Remote Manager** depuis le menu **Remote**.
