# AWS S3 und S3-kompatibel

RcloneView unterstützt **Amazon S3** und eine Vielzahl von **S3-kompatiblen Object-Storage-Anbietern**, darunter namhafte Anbieter wie Wasabi, Backblaze, Cloudflare R2 und weitere.

Diese Remotes teilen sich dieselbe Konfigurationsoberfläche, die auf dem S3-Protokoll basiert. Nach der Auswahl müssen Sie Ihre Zugangsdaten und optional eine benutzerdefinierte Endpoint-URL angeben.


## So fügen Sie einen S3-kompatiblen Remote hinzu

### Schritt 1: Öffnen des Fensters für die Konfiguration eines neuen Remotes

- Klicken Sie im oberen Menü unter **`Remote`** auf **`+ New Remote`**.
- Alternativ können Sie im Explorer-Bereich auf die Schaltfläche **`+`** klicken und **`New Remote`** auswählen.

### Schritt 2: Konfigurieren des S3-Anbieters

#### Im Tab **Provider**:
1. Suchen Sie nach `s3` oder dem Namen Ihres Anbieters (z. B. `aws`, `wasabi`, `minio`).
2. Wählen Sie **`S3`** aus der Ergebnisliste aus.

#### Im Tab **Options**:
3. Legen Sie Ihren **provider** fest (z. B. `AWS`, `Wasabi`, `Minio` usw.).
4. Geben Sie Ihre [**Access Key ID**](/howto/cloud-storage-setting/aws-account-info#step-by-step-get-your-aws-access-key-id-and-secret-access-key) ein.
5. Geben Sie Ihren [**Secret Access Key**](/howto/cloud-storage-setting/aws-account-info#step-by-step-get-your-aws-access-key-id-and-secret-access-key) ein.
6. Wählen Sie die passende **region** (z. B. `ap-northeast-2`) für Ihren Storage-Bucket aus.
   Legen Sie bei Bedarf einen benutzerdefinierten **endpoint** fest (für Anbieter, die nicht das AWS-Endpoint-Format verwenden).

### Schritt 3: Remote benennen und speichern

#### Im Tab **Name**:
7. Vergeben Sie einen Namen für den Remote (z. B. `myWasabi`, `minio-prod`).

#### Im Tab **Save**:
8. Klicken Sie auf **Save**, um die Einrichtung abzuschließen.
