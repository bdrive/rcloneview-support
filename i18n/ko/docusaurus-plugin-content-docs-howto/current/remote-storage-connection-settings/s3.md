# AWS S3 및 S3 호환 스토리지

RcloneView는 **Amazon S3**와 Wasabi, Backblaze, Cloudflare R2 등 주요 벤더를 포함한 다양한 **S3 호환 오브젝트 스토리지 프로바이더**를 지원합니다.

이러한 리모트는 S3 프로토콜을 기반으로 동일한 구성 인터페이스를 공유합니다. 선택 후에는 액세스 자격 증명과 필요에 따라 사용자 지정 엔드포인트 URL을 입력해야 합니다.


## S3 호환 리모트 추가 방법

### 1단계: 새 리모트 구성 창 열기

- 상단 메뉴의 **`Remote`** 아래에서 **`+ New Remote`**를 클릭합니다.
- 또는 Explorer 창의 **`+`** 버튼을 클릭하고 **`New Remote`**를 선택합니다.

### 2단계: S3 프로바이더 구성

#### **Provider** 탭에서:
1. `s3` 또는 사용 중인 프로바이더 이름(예: `aws`, `wasabi`, `minio`)을 검색합니다.
2. 결과 목록에서 **`S3`**를 선택합니다.

#### **Options** 탭에서:
3. **provider**를 설정합니다(예: `AWS`, `Wasabi`, `Minio` 등).
4. [**Access Key ID**](/howto/cloud-storage-setting/aws-account-info#step-by-step-get-your-aws-access-key-id-and-secret-access-key)를 입력합니다.
5. [**Secret Access Key**](/howto/cloud-storage-setting/aws-account-info#step-by-step-get-your-aws-access-key-id-and-secret-access-key)를 입력합니다.
6. 스토리지 버킷에 맞는 올바른 **region**(예: `ap-northeast-2`)을 선택합니다.
   필요한 경우 사용자 지정 **endpoint**를 설정합니다(AWS 엔드포인트 형식을 사용하지 않는 프로바이더의 경우).

### 3단계: 리모트 이름 지정 및 저장

#### **Name** 탭에서:
7. 리모트에 이름을 지정합니다(예: `myWasabi`, `minio-prod`).

#### **Save** 탭에서:
8. **Save**를 클릭하여 설정을 완료합니다.

2. 목록에 새 리모트가 표시되는지 확인합니다.
