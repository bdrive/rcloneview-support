# RcloneView 1.1 – 릴리스 노트

**릴리스 날짜:** 2025년 11월<br/>
**플랫폼:** Windows, macOS, Linux

RcloneView 1.1은 리모트 탐색, 터미널 액세스, 설정 관리, 인터페이스 동작 전반에 걸쳐 실용적인 새 기능과 사용성 개선 사항을 도입합니다. 이번 업데이트는 워크플로 효율성을 개선하고 사용자가 보고한 주요 문제를 해결하는 데 초점을 맞추고 있습니다.

## 새로운 기능

- **내장 터미널 탭:**
  이제 RcloneView 안에서 바로 사용할 수 있는 새로운 터미널 탭이 추가되었습니다. 더 이상 외부 터미널이나 명령 프롬프트로 전환할 필요 없이 앱에서 바로 셸 명령, rclone 명령, 간단한 진단을 실행할 수 있습니다.

- **앱 잠금(비밀번호 보호):**
  이제 RcloneView에 앱 수준의 비밀번호를 설정하여 무단 액세스를 방지할 수 있습니다.

- **사용자 지정 데이터베이스 위치 지원:**
  이제 `rclone_view.db` 파일의 저장 경로를 사용자가 직접 지정할 수 있어, 유연한 작업 환경 구성, 외장 드라이브 사용, 선택적 동기화가 가능합니다.

- **리모트 관리자: 목록 모드에서 항목 크기 표시:**
  이제 리모트 관리자의 목록 모드에서 각 항목의 상세 크기 정보를 확인할 수 있습니다.

- **"전체 경로 복사(리모트 포함)" 명령:**
  리모트 이름을 포함한 전체 경로를 클립보드에 편리하게 복사할 수 있는 기능이 추가되었습니다.

- **가상 리모트 폴더 트리 미리보기:**
  가상 리모트를 추가하거나 편집할 때 폴더 트리 미리보기가 표시되어 경로를 쉽게 선택할 수 있습니다.

## UI 및 UX 개선

- **반응형 탭 및 창 크기 조정:** 앱 창이 좁아지면 탭 크기가 동적으로 조정되어 텍스트가 잘리지 않습니다.
- **클라우드 폴더 탐색 수정:** iCloud Drive 및 유사한 클라우드 폴더가 이제 파일 탐색기에 올바르게 표시됩니다.
- **설정 패널 수정:** 로컬 rclone 실행 파일 경로가 더 이상 예기치 않게 사라지지 않습니다.
- **Combine 리모트 폴더 이름 필드:** Combine 유형 리모트가 이제 폴더 이름 입력을 위한 전용 텍스트 필드를 사용하여 명확성이 개선되었습니다.
- **전송 창 – 다크 모드 수정:** 테마 전환 시 발생하던 색상 불일치 문제를 해결했습니다.
- **히스토리 패널 개선:**
  - 오류 메시지가 더 이상 잘리지 않습니다.
  - 예약된 작업 오류가 이제 히스토리에 올바르게 표시됩니다.
- **탐색기 레이아웃 향상:** 탐색기 뷰 전반의 열 너비가 개선되어 가독성과 정렬 상태가 향상되었습니다.
- **지원되지 않는 리모트 유형 처리 개선:** 지원되지 않는 API를 사용하는 리모트 유형의 정보 대화 상자에 이제 적절한 대체 메시지가 표시됩니다.

## 버그 수정

- **비교 작업 UI 새로 고침 문제:** 비교 기반 복사 작업이 완료된 후에도 UI가 계속 새로 고침되던 버그를 수정했습니다.
- **동기화/비교 탐색기 수정:** 로컬 디스크 노드가 미리 선택된 상태에서 리모트 폴더 탐색이 실패하던 문제를 해결했습니다.
- **Windows 터미널 경로 처리:** Windows에서 터미널 환경 내에서 rclone 구성이 올바르게 읽히지 않던 문제를 수정했습니다.
- **작업 이름 유효성 검사 오류:** 작업 이름 고유성 검사 중 발생하던 데이터베이스 예외를 수정했습니다.
- **"변경 사항 없음" 성공 레이블 수정:** 동기화/비교 중 변경 사항이 감지되지 않은 경우 파일 상세 보기에 이제 "성공"이 올바르게 표시됩니다.
- **99% 완료 정확도 규칙:** 결과가 완전히 동일하지는 않지만 거의 동일한 경우, 진행률 표시기가 이제 잘못 100%로 표시되지 않고 99%로 표시됩니다.
