# RcloneView 1.0 – 릴리스 노트

**출시일:** 2025년 12월<br/>
**플랫폼:** Windows, macOS, Linux (Raspberry Pi OS 포함)

RcloneView 1.0은 향상된 안정성, 더 넓은 플랫폼 지원, 개선된 리모트 관리, 그리고 더 깔끔하고 정보성이 높아진 인터페이스와 함께 중요한 이정표를 세운 릴리스입니다. 이번 릴리스에는 사용자 피드백과 실제 사용 경험을 바탕으로 한 새로운 기능, UX 개선 사항, 주요 버그 수정이 포함되어 있습니다.

## 새로운 기능

- **Copy 작업 유형 지원:** Job Manager에 새로운 "Copy" 작업 유형이 추가되어 리모트 간 단방향 파일 전송을 더 쉽게 구성할 수 있습니다.
- **네트워크 드라이브 자동 감지 (Windows):** RcloneView가 이제 사용자 조작 없이 매핑된 네트워크 드라이브(예: SMB 공유)를 자동으로 감지하고 표시합니다. 이 드라이브들은 **Local** 탭에 바로 표시되어 동기화, 비교, 마운트에 즉시 사용할 수 있습니다.
- **컨텍스트 메뉴에서 크기 조회:** 선택한 폴더의 크기를 빠르게 계산할 수 있는 새로운 "Get Size" 옵션이 컨텍스트 메뉴에 추가되었습니다.
- **예정된 기능 패널:** 하단 로그 창에 계획 중이거나 곧 제공될 기능 목록을 보여주는 새로운 패널이 추가되어, 사용자가 향후 업데이트 소식을 쉽게 확인할 수 있습니다.

## UI 및 UX 개선

- **Transfer 창 재설계:** Transfer 작업 UI를 완전히 새롭게 개편하여 더 깔끔한 레이아웃을 제공하며, 성능과 가독성 향상을 위해 최대 100개의 항목을 표시합니다.
- **작업 상태 표시 개선:** 진행 중 및 완료된 작업을 더 정확하고 직관적으로 모니터링할 수 있도록 작업 상태 표시기를 개선했습니다.
- **Explorer 트리 동작 개선:** 트리 뷰가 최초 실행 시 루트 폴더를 자동으로 확장하여 탐색이 더 쉬워졌습니다. 또한 레이아웃 패널에서 폴더 이름이 겹쳐 보이던 문제를 해결했습니다.
- **마운트 UI 개선 (Windows/macOS):** Windows에서 로컬 폴더 선택 시 발생하던 레이아웃 문제를 수정했으며, 마운트/언마운트 시 Local Disk 탭이 자동으로 새로고침되도록 추가했습니다. 또한 macOS에서 마운트 경로 업데이트 동작 및 다중 리모트 `cmount` 문제를 수정했습니다.
- **리모트 선택 및 표시 수정:** 동기화 및 비교 작업 중 클라우드 드라이브가 올바르게 표시되도록 리모트 선택 대화상자를 개선했습니다.

## 버그 수정

- **Synology NAS 자동 감지 수정:** 잘못된 감지 문제를 해결하고 NAS 리모트 추가 시 기본 주소가 저장되도록 보장했습니다.
- **재시작 시 기본 리모트 재설정 문제:** 추가 창을 사용한 후 앱을 재시작하면 기본 리모트 연결이 초기화되던 문제를 수정했습니다.
- **Jobs 탭 표시 문제:** Jobs 탭에서 파일 및 속도 정보가 누락되던 문제를 해결했습니다.
- **AWS S3 동작 수정:** 동기화 오류를 방지하기 위해 `--no-check-bucket`을 기본값으로 추가했습니다.
- **Raspberry Pi OS 12 호환성:** 이제 RcloneView가 Raspberry Pi OS 12에서 정상적으로 실행됩니다.
