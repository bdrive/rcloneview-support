# RcloneView 1.3 – 릴리스 노트

**릴리스 날짜:** 2026년 2월<br/>
**플랫폼:** Windows, macOS, Linux

RcloneView 1.3에서는 새로운 작업 관리 기능이 추가되었으며 Slack, Discord와 같은 인기 커뮤니케이션 도구와의 연동이 확장되었습니다. 또한 이메일 알림, UI 동작, 작업 실행 추적 기능이 개선되었습니다.

## 새로운 기능

- **배치 작업 지원 (베타):**
  이제 여러 작업을 배치로 그룹화하여 함께 실행할 수 있습니다.
  이 베타 기능을 사용하면 순서대로 실행할 작업 시퀀스를 정의하여 관련 작업을 더 쉽게 관리할 수 있습니다.
  추가 제어 기능과 오류 처리는 향후 업데이트에서 제공될 예정입니다.

- **이동(Move) 작업 유형 추가:**
  파일을 소스에서 대상으로 이동하여 전송할 수 있는 새로운 "이동" 작업 유형이 추가되었습니다.

- **실패한 작업 재시도:**
  이제 실패한 작업을 수동으로 다시 만들지 않고도 재시도할 수 있습니다.

- **Slack 연동:**
  RcloneView가 이제 Slack 알림과 Slack 채널을 통한 작업 제어를 지원합니다.

- **Discord 연동:**
  이제 Discord를 통해 작업 알림을 받을 수 있습니다.

- **원격 제어 설정 가이드 위치 조정:**
  Telegram 및 Slack 원격 제어 설정 가이드 링크를 연동 섹션 상단에서 더 쉽게 찾을 수 있습니다.

- **인도네시아어 지원:**
  이제 인터페이스를 Bahasa Indonesia로 사용할 수 있습니다.

## 개선 사항

- **이메일 알림 개선:**
  - 이메일 설정 패널의 레이아웃과 가독성 개선
  - 작업 상태 이메일에 상세 정보 포함
  - 이메일 설정 화면에서 드물게 발생하던 UI 충돌 수정
  - 무료 체험 종료 후에도 이메일 알림을 계속 사용할 수 있도록 개선

- **리모트 아이콘 업데이트:** 리모트를 더 쉽게 식별할 수 있도록 새로운 아이콘을 추가하고 기존 아이콘을 새로 고쳤습니다.

- **파일 목록 로딩 안정성 개선:** 파일 로딩이 느릴 때 다른 화면으로 이동하면 오래된 내용이 표시되던 문제를 방지했습니다.

- **"변경 사항 없음" 결과 처리:** 작업 완료 상태와 관계없이 상세 보기에서 "변경 사항 없음" 상태가 유지되도록 개선했습니다.

## 버그 수정

- **OneDrive 파일 삭제:** OneDrive 리모트에서 파일 삭제가 되지 않던 문제를 해결했습니다.
- **전송 상태 표시 버그:** 작업 완료 후에도 파일이 계속 전송 중인 것처럼 표시되던 문제를 수정했습니다.
- **로컬 디스크에서 해시로 비교:** 로컬 스토리지에서 해시 확인으로 파일을 비교할 때 발생하던 멈춤 현상을 수정했습니다.
- **작업 기록 부정확 문제:** 강제 종료된 작업이 기록(History)에서 "실행 중" 상태로 남아 있던 문제를 수정하여 이제 올바르게 제거됩니다.
- **전송 취소 툴팁 정렬 오류:** 전송 화면에서 툴팁 위치가 어긋나던 문제를 수정했습니다.
- **동일한 이름의 리모트 재등록:** 이전과 같은 이름으로 리모트를 추가할 때 오래된 파일 목록이 재사용되던 문제를 방지했습니다.
- **드라이 런 열 잘림 현상:** 드라이 런 상세 패널의 마지막 열이 이제 내용에 맞게 자동으로 확장됩니다.
- **외부 rclone + Google Drive 토큰 오류:** 헤드리스 모드 Google Drive 토큰과 함께 외부 rclone을 사용할 때 발생하던 연결 문제를 수정했습니다.
