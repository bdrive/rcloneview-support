# RcloneView 1.4 – Notas de la versión

**Fecha de lanzamiento:** marzo de 2026<br/>
**Plataformas:** Windows, macOS, Linux

RcloneView 1.4 introduce la integración con iCloud Photos, un frontend web para acceso remoto, capacidades mejoradas de Batch Job, y numerosas mejoras de estabilidad y usabilidad en todas las plataformas.

## Aspectos destacados

- **Soporte para iCloud Photos (Beta, solo lectura):**
  Explora y descarga fotos de tu biblioteca de iCloud Photos directamente dentro de RcloneView.
  Visualiza álbumes, busca por fecha y descarga originales, todo sin salir de la aplicación.
  Esta función beta admite acceso de solo lectura: subir, eliminar y renombrar no están disponibles por el momento.

- **Frontend web (Beta):**
  Accede a RcloneView desde cualquier lugar a través de tu navegador. El servidor web integrado ofrece
  un explorador de archivos de doble panel con todas las funciones y monitoreo de transferencias en tiempo real.
  Incluye acceso protegido por contraseña y una opción para permitir conexiones remotas.

- **Batch Job mejorado:**
  Los Batch Jobs se han mejorado significativamente con ejecución paso a paso, dry run por paso,
  dependencias de variables entre pasos, indicadores de nivel de riesgo, búsqueda de plantillas y seguimiento completo del historial de ejecución.
  Construye flujos de trabajo complejos de varios pasos con confianza gracias a las vistas previas de dry run y una clara visibilidad de riesgos.

## Nuevas funciones

- **Guardar y cargar preajustes de filtros:**
  Guarda y carga preajustes de filtros para reutilizarlos en diferentes trabajos y sesiones.

- **Opciones de montaje en Windows:**
  La configuración de permisos de archivos y directorios ahora está disponible en Windows para operaciones de montaje.

## Mejoras

- **Soporte para macOS 12:**
  Se restauró la compatibilidad con macOS 12 (Monterey).

- **Exclusión de archivos ocultos de Dropbox:**
  Opción para excluir automáticamente los archivos ocultos (por ejemplo, .DS_Store) al sincronizar con Dropbox.

- **Compatibilidad del icono de bandeja en Linux:**
  Se mejoró el manejo de la bandeja del sistema para Linux Cinnamon y otros escritorios con soporte nativo de bandeja.

- **Color del icono de bandeja en Windows:**
  El color del icono de bandeja ahora es ajustable en Windows.

- **Manejo de la ruta de la base de datos:**
  Se mejoró la configuración de la ruta de la base de datos con soporte de ruta heredada para actualizaciones más fluidas.

- **Proton Drive eliminado:**
  Proton Drive se eliminó de la lista de proveedores de remotos debido a cambios en el proveedor original.

## Correcciones de errores

- **Flags personalizados no aplicados:**
  Se corrigió un problema por el cual los flags personalizados se ignoraban durante la ejecución de trabajos.

- **Dry Run en trabajos de tipo Move:**
  Se corrigió que el botón Dry Run no estuviera habilitado para trabajos de tipo Move.

- **Estado maximizado de la terminal:**
  Se corrigieron dos problemas cuando el panel de terminal estaba maximizado, incluidos errores de estado de elementos de la interfaz.

- **Inicialización de la base de datos:**
  Se eliminaron llamadas duplicadas a sqfliteFfiInit() que podían causar errores intermitentes de "base de datos cerrada".

- **ID de dispositivo en almacenamiento seguro:**
  Se corrigió el manejo del ID de dispositivo en el almacenamiento seguro y se resolvió un error por el cual el estado deshabilitado de la autodetección de Synology no se guardaba.

- **Retraso en la entrada del programador:**
  Se corrigió el retraso de la interfaz al ingresar parámetros de programación.
