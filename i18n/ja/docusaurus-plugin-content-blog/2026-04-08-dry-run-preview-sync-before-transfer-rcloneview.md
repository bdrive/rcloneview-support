# RcloneViewで転送前にドライランで同期の変更をプレビューする

> 一つの設定ミスの同期が、数秒で数千のファイルを削除してしまうことがあります。**RcloneView**では、1バイトも転送する前にドライランですべての変更をプレビューでき、同期を実行する前に完全な確信を持つことができます。

同期操作はrcloneの最も強力な機能の一つです。送信先を送信元と一致させるため、新しいファイルを転送し、変更されたファイルを更新し、送信元に存在しなくなったファイルを削除します。この最後の部分、削除こそが、同期を強力かつ危険なものにしています。

ドライランは、実際にファイルを移動、コピー、削除することなく、同期操作全体をシミュレートします。何が起こるかを正確に示してくれます。どのファイルが転送されるか、どのファイルが削除されるか、どのファイルがスキップされるかです。出力内容を確認し、期待通りであることを確かめてから、初めて実際の同期を実行します。

RcloneViewはドライランを同期ワークフローに直接統合しており、GUIを通じて変更を簡単にプレビューしてから実行できます。2つのクラウドリモート間で同期する場合でも、ローカルファイルをクラウドにバックアップする場合でも、毎回最初のステップとしてドライランを行うべきです。

## ドライランが行うこと

ドライランモードを有効にすると、rcloneは実際の同期と同じ解析をすべて実行します。送信元と送信先をスキャンし、サイズ、タイムスタンプ、またはチェックサムでファイルを比較し、実行する操作のリストを作成します。唯一の違いは、これらの操作が実際には実行されないという点です。

ドライランの出力は以下を教えてくれます。
- **転送されるファイル**: 送信元から送信先にコピーされる、新規または変更されたファイル。
- **削除されるファイル**: 送信元に存在せず、削除される送信先のファイル。
- **スキップされるファイル**: 両方で同一であり、操作が不要なファイル。
- **合計データ量**: どれだけのデータが転送されるか。時間や帯域幅の要件を見積もるのに役立ちます。

このシミュレーションは読み取り専用です。ファイルの移動、コピー、削除は一切行われません。送信元と送信先は完全に手つかずのまま残ります。

## 破壊的な操作においてドライランが重要な理由

`sync`コマンドは、送信元に存在しないファイルを送信先で削除するため、本質的に破壊的です。これは設計上の仕様であり、同期がコピーと異なる点です。しかし、それはミスが結果を招くことも意味します。

- **同期方向の誤り**: 誤って送信元と送信先を入れ替えてしまうと、同期は空または古い送信先に合わせて送信元のファイルを削除してしまいます。
- **パスの誤り**: 誤ったディレクトリに同期すると、無関係のファイルが上書きされたり、大量削除が引き起こされたりする可能性があります。
- **フィルターの設定ミス**: include/excludeフィルターが誤っていると、保持するつもりだったファイルが送信元スキャンから除外され、その結果として送信先で削除されてしまう可能性があります。
- **認証情報の失効**: 送信元リモートの認証情報が期限切れになっている場合、空に見えてしまい、同期が送信先のすべてを削除してしまう可能性があります。

ドライランは、これらすべての問題を実際の損害が発生する前に発見してくれます。出力を確認するのにかかる数秒は、復旧作業の何時間もの節約になります。

## RcloneViewでドライランを有効にする方法

RcloneViewには、同期のプレビューを実行する簡単な方法があります。

1. **ジョブマネージャー**を開き、プレビューしたい同期ジョブを選択します。
2. ジョブ設定の**カスタムフラグ**セクションに`--dry-run`フラグを追加します。
3. ジョブを実行します。RcloneViewが同期をシミュレートし、結果を表示します。
4. 転送モニターで出力を確認し、何が起こるかを確認します。
5. すべて問題なければ、`--dry-run`フラグを削除し、実際にジョブを実行します。

または、組み込みのターミナルを使ってドライランコマンドを直接実行することもできます。
```
rclone sync source: dest: --dry-run
```

## ドライランの出力の読み方

ドライランの出力は実際の同期と同じ形式を使いますが、操作は実行されません。確認すべきポイントは以下の通りです。

**転送されるファイル**は、パスとサイズとともにリストされます。これらが更新または追加を期待しているファイルであることを確認してください。すでに同期されているはずのファイルが転送対象として表示される場合は、タイムスタンプの不一致やチェックサムの差異を調査する価値があるかもしれません。

**削除されるファイル**は、最も重要に確認すべき項目です。削除対象としてリストされた各ファイルは、実際の同期時に送信先から完全に削除されます。保持したいファイルがここに表示されている場合は、送信元パス、フィルター、または同期方向の調整が必要かもしれません。

**スキップされるファイル**は、すでに同期されているファイルが正しく識別されていることを確認します。健全なドライラン出力は、ほとんどがスキップされるファイルで、転送と削除は少数であるべきです。

最後の**サマリー統計**は、転送数、削除数、および全体のデータ量を教えてくれます。これらの数値を期待値と比較してください。1つのファイルだけを変更したフォルダを同期しているのに、ドライランが数千件の転送を示している場合は、何かの設定に誤りがあります。

## ドライランが発見した実際の予期せぬ事態

以下は、ドライランがデータ損失を防いだ実際のシナリオです。

**予期しない大量削除**: Google DriveからS3への同期を設定しましたが、Google Driveのトークンが期限切れになっていました。rcloneは送信元が空だと認識し、送信先のすべてを削除しようとします。ドライランは数千件の削除とゼロ件の転送を表示し、即座に問題を発見できます。

**フィルターの不一致**: `*.tmp`ファイルの除外フィルターを追加しようとしましたが、誤って`*.tmpl`と入力してしまい、これがTerraformテンプレートファイルにマッチしてしまいました。ドライランは、これらのテンプレートファイルが送信先から削除されることを表示し、タイプミスに気づかせてくれます。

**ベースディレクトリの誤り**: `remote:projects/2026`を同期するつもりでしたが、`remote:projects`と入力してしまいました。ドライランは、すべてのプロジェクト年のファイルが同期されることを明らかにし、意図していなかったサブディレクトリのファイルが上書きされる可能性を示してくれます。

**大文字小文字の区別の問題**: 大文字小文字を区別しないリモート(OneDriveなど)と大文字小文字を区別するリモート(S3など)の間でファイルを移動すると、重複ファイルが作成される可能性があります。ドライランはこれらの予期しない転送を表示するので、大文字小文字の競合が増える前に対処できます。

## ドライランをワークフローの一部にする

最大限の安全性を確保するために、ドライランを標準的な運用手順に組み込みましょう。

**手動同期の場合**: 同期操作を実行する前に、必ず最初にドライランを実行してください。出力を確認してから、フラグを削除して実際の同期を実行します。この余分な1分は、心の安らぎに値する価値があります。

**新しいスケジュールジョブの場合**: 新しいスケジュール同期を作成する際は、まず`--dry-run`を付けて手動で実行してください。ドライランの出力が期待通りであることを確認してから、スケジュールを有効にしましょう。

**設定変更後**: ジョブの送信元、送信先、フィルター、フラグを変更するたびに、次回実行の前にドライランを実行してください。設定変更は、ドライランで明らかになる予期しない副作用を引き起こす可能性があります。

**既存ジョブの定期確認**: 安定して長期間稼働しているスケジュールジョブであっても、時折ドライランでの確認を行うことに価値があります。送信元データ、リモート設定、またはプロバイダーの動作の変化は、時間とともに同期の動作を変える可能性があります。

## ドライランと比較機能の組み合わせ

RcloneViewのフォルダ比較機能は、送信元と送信先の内容を視覚的に並べて表示することで、ドライランを補完します。両方を併用することで、同期前の包括的な可視性が得られます。

1. **比較**機能を使って、送信元と送信先の差異を視覚的に確認します。
2. **ドライラン**を実行し、その差異に対して同期操作が実際に何を行うかを正確に確認します。
3. 計画された操作が、比較で確認した内容と一致していることを確認します。
4. 確信を持って同期を実行します。

<img src="/support/images/en/howto/rcloneview-basic/compare-display-select.png" alt="Compare shared folder and My Drive contents" class="img-large img-center" />

比較機能は現在の状態を示し、ドライランは計画された動作を示します。両者を組み合わせることで、推測に頼ることなく、何が異なっているのか、そしてそれに対して何が行われるのかを理解できます。

## 応用編: --dry-runを他のフラグと組み合わせて使う

ドライランは他のすべてのrcloneフラグと併用できるため、実際の本番設定を正確にシミュレートできます。

- `--dry-run --backup-dir remote:backup`は、同期内容とバックアップコピーの保存先の両方をプレビューします。
- `--dry-run --filter-from filters.txt`は、フィルタールールが期待通りの結果を生成することを検証します。
- `--dry-run --max-age 24h`は、過去24時間以内に変更されたファイルのみを同期する動作をシミュレートします。
- `--dry-run -v`は、計画された各操作についての詳細を示す冗長な出力を追加します。

この組み合わせ可能性により、本番環境にデプロイする前に、あらゆる設定を安全にテストできます。

## はじめに

1. [rcloneview.com](https://rcloneview.com/src/download.html)から**RcloneViewをダウンロード**します。
2. 希望の送信元と送信先で同期ジョブを作成します。
3. カスタムフラグに`--dry-run`を追加し、ジョブを実行して変更をプレビューします。
4. 出力を注意深く確認してから、フラグを削除して実際の同期を実行します。

ドライランは無料で、数秒しかかからず、壊滅的なデータ損失を防ぐことができます。すべての同期操作の最初のステップにすれば、同期がファイルに対して行うことに驚かされることは二度とないでしょう。
