# AWS S3 および S3互換ストレージ

RcloneViewは、**Amazon S3**、および Wasabi、Backblaze、Cloudflare R2 などの主要ベンダーを含む幅広い**S3互換オブジェクトストレージプロバイダー**をサポートしています。

これらのリモートは、S3プロトコルに基づいた同じ設定インターフェースを共有します。選択後、アクセス認証情報と、必要に応じてカスタムエンドポイントURLを入力します。


## S3互換リモートの追加方法

### ステップ1: 新規リモート設定ウィンドウを開く

- 上部メニューの **`Remote`** から **`+ New Remote`** をクリックします。
- または、エクスプローラーペインの **`+`** ボタンをクリックし、**`New Remote`** を選択します。

### ステップ2: S3プロバイダーの設定

#### **Provider** タブで:
1. `s3` またはお使いのプロバイダー名（例: `aws`、`wasabi`、`minio`）を検索します。
2. 結果リストから **`S3`** を選択します。

#### **Options** タブで:
3. **provider**（例: `AWS`、`Wasabi`、`Minio` など）を設定します。
4. [**Access Key ID**](/howto/cloud-storage-setting/aws-account-info#step-by-step-get-your-aws-access-key-id-and-secret-access-key) を入力します。
5. [**Secret Access Key**](/howto/cloud-storage-setting/aws-account-info#step-by-step-get-your-aws-access-key-id-and-secret-access-key) を入力します。
6. ストレージバケットに対応する正しい **region**（例: `ap-northeast-2`）を選択します。
   必要に応じて、カスタム **endpoint**（AWSのエンドポイント形式を使用しないプロバイダー向け）を設定します。

### ステップ3: リモートに名前を付けて保存

#### **Name** タブで:
7. リモートに名前を付けます（例: `myWasabi`、`minio-prod`）。

#### **Save** タブで:
8. **Save** をクリックして設定を完了します。

1. **Remote** メニューから **Remote Manager** を開きます。
2. リストに新しいリモートが表示されていることを確認します。
