# Storj 白标探索器（预览版）

本页面是 RcloneView 如何以完全品牌化的 **Storj Explorer** 桌面应用形式交付给您的用户的私密预览。

此白标方案的目标是：

- 让 **Storj 品牌贯穿整个产品**，处于最显眼的位置。
- 让用户在安装后**能够轻松连接其 Storj 账户**。
- 确保 **Storj 在每个管理和导航流程中都是首选**。
- 为 Storj 品牌部署提供**优先支持与维护**服务。

---

## 1. 品牌套件与视觉整合

我们提供的品牌套件将 Storj 作为整个产品中主要且可见的品牌。所有 RcloneView 品牌标识均被移除，或仅在法律要求的情况下保留（例如内部版本字符串）。

主要元素：

- 应用名称设置为 **“Storj Explorer”**（或其他商定的名称）。
- 所有 RcloneView 徽标均替换为 **Storj 徽标**：
  - 桌面快捷方式和任务栏图标。
  - 安装程序图标。
  - 应用内标题栏及窗口图标。
- 在合适之处调整颜色强调色，以匹配 Storj 的品牌配色方案。


<img src="/support/images/en/howto/remote-storage-connection-settings/storj-brandkit-example.png" alt="storj brandkit example" class="img-large img-center" />

## 2. 安装后 Storj 远程向导

安装完成后，系统会立即引导用户连接其 Storj 账户，使他们无需额外设置步骤即可开始使用该服务。

主要行为：

- 在安装向导结束时，启动应用会打开 **“添加 Storj 远程”** 作为默认的引导流程。
- 该向导已针对 Storj 进行简化并预配置：
  - 提供商类型预先选定为 **Storj**。
  - 默认仅显示 Storj 专属选项。
  - 高级选项仍可通过 **“显示高级选项”** 链接查看。

<img src="/support/images/en/howto/remote-storage-connection-settings/post-install-storj-remote-wizard.png" alt="post install storj remote wizard" class="img-large img-center" />

### 主页快速访问向导

如果用户跳过初始连接或关闭向导：

- 主页右侧面板会显示一个**大号 Storj 图块**。
- 点击该图块可随时重新打开 **“添加 Storj 远程”** 向导。

<img src="/support/images/en/howto/remote-storage-connection-settings/on-home-quick-access-wizard-for-storj.png" alt="on home quick access wizard for storj" class="img-large img-center" />

这确保了对于新用户或回访用户来说，连接 Storj 始终是最显眼的下一步操作。

---

## 3. Storj 优先的导航与管理

一旦添加了 Storj 远程，UI 便会进行优化，使 Storj 在所有关键导航和管理界面中始终保持突出可见。

### 3.1 重新启动时固定 Storj 面板

在配置了 Storj 远程之后：

- 在后续启动时，UI 会默认打开并**固定 Storj 面板**。
- 典型布局：
  - 左侧：本地磁盘或其他来源。
  - 右侧：用户的 **MYStorj** 远程。
- 用户可以通过拖放自由重新排列面板，因此无论 Storj 面板当前位于哪一侧，都可以在左右两侧之间移动。
- 无论 Storj 面板当前处于哪一侧，这都使本地文件夹与 Storj 之间的重复同步或复制操作变成一键式的操作。

<img src="/support/images/en/howto/remote-storage-connection-settings/storj-panel-pinned-on-re-launch.png" alt="storj panel pinned on re launch" class="img-large img-center" />


### 3.2 在“新建远程”和“远程管理器”中优先显示 Storj

为了突出 Storj 作为主要存储提供商：

- 在**新建远程**对话框中：
  - Storj 显示在**提供商列表的顶部**。
  - Storj 图块以视觉方式高亮显示，以鼓励用户选择。
- 在**远程管理器**中：
  - Storj 远程（例如 `MYStorj`）显示在**远程列表的顶部**。
  - 在列表视图和卡片视图中，Storj 远程均以视觉方式突出显示，便于更快访问。



<img src="/support/images/en/howto/remote-storage-connection-settings/storj-first-in-management-dialog.png" alt="storj first in management dialog" class="img-large img-center" />

## 4. Storj 专属优先支持与维护

针对 Storj 白标部署，我们提供专属的支持与维护服务。

包含的服务：

- **专属文档**  
  - 专门为 **Storj Explorer** 用户提供的独立手册页面。  
  - 关于使用 Storj 进行连接、同步和故障排查的分步指南。

- **优先事件处理**  
  - Storj 用户的问题在支持队列中会被优先处理。  
  - 针对影响 Storj 用户的重大事件（例如连接故障、数据访问问题）提供**紧急响应**。

- **持续产品更新**  
  - 作为维护协议的一部分，定期提供桌面客户端升级。  
  - 在双方共同批准后，能够以 Storj 品牌形式推出新的 RcloneView 功能。

## 5. 后续步骤

如果您希望推进该白标产品的落地，请按以下步骤进行：

1. **品牌一致性确认**
   - 确认 Storj 徽标使用规范和品牌配色。
   - 确定最终产品名称（例如 *Storj Explorer*）。
2. **体验定义**
   - 验证上述引导流程和 Storj 优先行为。
   - 调整任何默认设置（例如默认同步模式、默认挂载路径）。
3. **试点构建**
   - 我们将交付一个私密试点版本以及针对 Storj 的专属文档，供内部测试使用。
