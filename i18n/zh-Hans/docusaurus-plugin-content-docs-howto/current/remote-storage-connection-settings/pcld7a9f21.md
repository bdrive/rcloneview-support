# pCloud 白标资源管理器（预览版）

本页面是 RcloneView 如何以完全品牌化的 **pCloud Explorer** 桌面应用形式向您的用户交付的私密预览。

此白标方案的目标是：

- 让 **pCloud 品牌贯穿整个产品**，处于核心位置。
- 让用户在安装后**能够毫不费力地连接其 pCloud 账户**。
- 确保 **pCloud 成为每个管理和导航流程中的首选**。
- 为 pCloud 品牌化部署提供**优先支持和维护**。

---

## 1. 品牌套件与视觉集成

我们提供一套品牌套件，使 pCloud 成为产品中主要且可见的品牌。所有 RcloneView 品牌标识均被移除，或仅在法律要求的地方保留（例如内部版本字符串）。

关键元素：

- 应用程序名称设置为 **“pCloud Explorer”**（或另行商定的名称）。
- 所有 RcloneView 徽标替换为 **pCloud 徽标**：
  - 桌面快捷方式和任务栏图标。
  - 安装程序图标。
  - 应用内标题栏和窗口图标。
- 在适当的情况下调整颜色以匹配 pCloud 的品牌配色方案。

<img src="/support/images/en/howto/remote-storage-connection-settings/pcloud-brandkit-examples.png" alt="pcloud brandkit examples" class="img-large img-center" />


---

## 2. 安装后 pCloud 远程向导

安装完成后，系统会立即引导用户连接其 pCloud 账户，使其无需额外的设置步骤即可开始使用该服务。

主要行为：

- 在安装向导结束时，启动应用会打开 **“添加 pCloud 远程”** 作为默认的入门引导流程。
- 该向导已针对 pCloud 进行了简化和预配置：
  - 提供商类型预先选定为 **pCloud**。
  - 默认仅显示 pCloud 专属选项。
  - 高级选项仍可通过 **“显示高级选项”** 链接访问。

<img src="/support/images/en/howto/remote-storage-connection-settings/post-install-pcloud-remote-wizard.png" alt="post install pcloud remote wizard" class="img-large img-center" />

### 主页快速访问向导

如果用户跳过初始连接或关闭了向导：

- 主屏幕右侧面板会显示一个**大型 pCloud 图块**。
- 点击该图块可随时重新打开 **“添加 pCloud 远程”** 向导。

<img src="/support/images/en/howto/remote-storage-connection-settings/on-home-quick-access-wizard.png" alt="on home quick access wizard" class="img-large img-center" />


这确保连接 pCloud 始终是新用户或回访用户最显眼的下一步操作。

---

## 3. pCloud 优先的导航与管理

一旦添加了 pCloud 远程，UI 便会进行优化，以在所有关键导航和管理界面中保持 pCloud 的醒目可见性。

### 3.1 RcloneView 资源管理器中的 pCloud 驱动器

当 pCloud 远程作为本地驱动器挂载时：

- pCloud 驱动器（例如 `pCloud Drive (P:/)`）会显示在 **RcloneView 本地远程列表的顶部**。
- 该驱动器使用 **pCloud 图标**，以便与其他驱动器在视觉上区分开来。

<img src="/support/images/en/howto/remote-storage-connection-settings/pcloud-first-in-explorer.png" alt="pcloud first in explorer" class="img-medium img-center" />

### 3.2 重新启动时固定的 pCloud 面板

配置好 pCloud 远程后：

- 在后续启动时，UI 默认会打开一个**固定的 pCloud 面板**。
- 典型布局：
  - 左侧：本地磁盘或其他来源。
  - 右侧：用户的 **MYpCloud** 远程。
- 这使得本地文件夹与 pCloud 之间的重复同步或复制操作只需一键即可完成。

<img src="/support/images/en/howto/remote-storage-connection-settings/pcloud-panel-pinned-on-re-launch.png" alt="pcloud panel pinned on re launch" class="img-large img-center" />


### 3.3 “新建远程”和“远程管理器”中的 pCloud 优先显示

为突出 pCloud 作为主要存储提供商的地位：

- 在**新建远程**对话框中：
  - pCloud 出现在**提供商列表的顶部**。
  - pCloud 图块在视觉上被高亮显示，以鼓励用户选择。
- 在**远程管理器**中：
  - pCloud 远程（例如 `MYpCloud`）显示在**远程列表的顶部**。
  - 在列表视图和卡片视图中，pCloud 远程均在视觉上被强调，以便更快访问。


<img src="/support/images/en/howto/remote-storage-connection-settings/pcloud-first-in-management-dialog.png" alt="pcloud first in management-dialog" class="img-large img-center" />

## 4. pCloud 优先支持与维护

针对 pCloud 白标部署，我们提供专门的支持与维护服务线。

包含的服务：

- **专属文档**  
  - 专门为 **pCloud Explorer** 用户编写的独立说明手册页面。  
  - 关于连接、同步以及 pCloud 相关问题排查的分步指南。

- **优先事件处理**  
  - pCloud 用户的问题会在支持队列中被优先处理。  
  - 针对影响 pCloud 用户的严重事件（例如连接失败、数据访问问题）提供**紧急响应**。

- **持续的产品更新**  
  - 维护协议中包含定期的桌面客户端升级。  
  - 经双方共同批准后，能够以 pCloud 品牌形式推出 RcloneView 的新功能。

## 5. 后续步骤

如果您希望推进此白标产品：

1. **品牌一致性确认**
   - 确认 pCloud 徽标使用规范和品牌配色。
   - 确定最终产品名称（例如 *pCloud Explorer*）。
2. **体验定义**
   - 验证上述入门引导流程和 pCloud 优先行为。
   - 调整任何默认设置（例如默认同步模式、默认挂载路径）。
3. **试点构建**
   - 我们将提供一个私密试点版本和 pCloud 专属文档，供内部测试使用。
