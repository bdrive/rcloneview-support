# RcloneView 1.0 – 发行说明

**发布日期：** 2025 年 12 月<br/>
**支持平台：** Windows、macOS、Linux(包括 Raspberry Pi OS)

RcloneView 1.0 是一个重要里程碑,带来了增强的稳定性、更广泛的平台支持、改进的远程管理,以及更简洁、信息更丰富的界面。此版本基于用户反馈和实际使用情况,包含了新功能、用户体验改进以及关键错误修复。

## 新功能

- **支持复制作业类型:** 在作业管理器中新增了“复制”(Copy)作业类型,使配置远程之间的单向文件传输更加便捷。
- **自动网络驱动器检测(Windows):** RcloneView 现在无需用户操作即可自动检测并显示已映射的网络驱动器(例如 SMB 共享)。这些驱动器会直接显示在**本地**(Local)选项卡中,方便用于同步、比较和挂载。
- **右键菜单中的“获取大小”功能:** 右键菜单中新增了“获取大小”(Get Size)选项,可快速计算所选文件夹的大小。
- **即将推出的功能面板:** 底部日志窗口中新增了一个面板,用于显示计划中和即将推出的功能列表,帮助用户及时了解未来的更新。

## 界面与用户体验改进

- **传输窗口重新设计:** 全面改版了传输(Transfer)作业界面,布局更简洁,最多可显示 100 条记录,以提升性能和可读性。
- **作业状态显示优化:** 改进了作业状态指示器,可更准确、直观地监控进行中和已完成的任务。
- **资源管理器树形视图行为:** 树形视图现在会在首次启动时自动展开根文件夹,以便于导航。同时解决了布局窗格中文件夹名称重叠的问题。
- **挂载界面增强(Windows/macOS):** 修复了在 Windows 上选择本地文件夹时的布局问题,并新增了挂载/卸载时自动刷新本地磁盘选项卡的功能。同时修复了挂载路径更新行为以及 macOS 上多远程 `cmount` 的问题。
- **远程选择与显示修复:** 改进了远程选择对话框,使其在同步和比较操作期间能正确显示云端驱动器。

## 错误修复

- **Synology NAS 自动检测修复:** 解决了检测不正确的问题,并确保在添加 NAS 远程时保存默认地址。
- **重启后默认远程被重新分配:** 修复了在使用额外窗口后重启应用会重置默认远程连接的问题。
- **作业选项卡显示问题:** 解决了作业(Jobs)选项卡中文件和速度信息缺失的问题。
- **AWS S3 行为修复:** 默认添加了 `--no-check-bucket`,以防止同步错误。
- **Raspberry Pi OS 12 兼容性:** RcloneView 现在可以在 Raspberry Pi OS 12 上正常启动和运行。
