# Storj 白牌探索工具（預覽版）

本頁面為私人預覽內容，展示 RcloneView 如何以完全客製品牌的 **Storj Explorer** 桌面應用程式形式，提供給您的使用者。

此白牌提案的目標為：

- 讓 **Storj 品牌成為整個產品的核心焦點**。
- 讓使用者在安裝完成後**能夠輕鬆連接其 Storj 帳戶**。
- 確保 **Storj 在每個管理與導覽流程中都是首選**。
- 為 Storj 品牌部署**提供專屬的優先支援與維護服務**。

---

## 1. 品牌套件與視覺整合

我們提供一套品牌套件，讓 Storj 成為產品中主要且顯眼的品牌。所有 RcloneView 的品牌元素都會被移除，或僅保留法律要求的部分（例如內部版本字串）。

主要元素：

- 應用程式名稱設定為「**Storj Explorer**」（或其他協議名稱）。
- 所有 RcloneView 標誌皆替換為 **Storj 標誌**：
  - 桌面捷徑與工作列圖示。
  - 安裝程式圖示。
  - 應用程式內標題列與視窗圖示。
- 依需求調整色彩配置，使其符合 Storj 的品牌色彩。

## 2. 安裝後 Storj 遠端精靈

安裝完成後，系統會立即引導使用者連接其 Storj 帳戶，讓他們無需額外設定步驟即可開始使用服務。

主要行為：

- 在安裝精靈結束時，啟動應用程式會開啟「**新增 Storj 遠端**」作為預設的導覽流程。
- 精靈流程已針對 Storj 進行簡化與預先設定：
  - 供應商類型預先選取為 **Storj**。
  - 預設僅顯示 Storj 專屬選項。
  - 進階選項仍可透過「**顯示進階選項**」連結存取。

<img src="/support/images/en/howto/remote-storage-connection-settings/post-install-storj-remote-wizard.png" alt="post install storj remote wizard" class="img-large img-center" />

### 首頁快速存取精靈

若使用者略過初次連接或關閉精靈：

- 首頁右側面板會顯示一個**大型 Storj 圖磚**。
- 點擊該圖磚可隨時重新開啟「**新增 Storj 遠端**」精靈。

<img src="/support/images/en/howto/remote-storage-connection-settings/on-home-quick-access-wizard-for-storj.png" alt="on home quick access wizard for storj" class="img-large img-center" />

這確保了連接 Storj 對新使用者或回訪使用者而言，始終是最顯眼的下一步操作。

---

## 3. Storj 優先的導覽與管理

一旦新增 Storj 遠端後，UI 便會經過最佳化，使 Storj 在所有主要導覽與管理畫面中保持顯著可見。

### 3.1 重新啟動時固定顯示 Storj 面板

在設定完 Storj 遠端後：

- 後續啟動時，UI 會預設以**固定顯示的 Storj 面板**開啟。
- 典型版面配置：
  - 左側：本機磁碟或其他來源。
  - 右側：使用者的 **MYStorj** 遠端。
- 使用者可自由透過拖放調整面板配置，因此 Storj 面板可依偏好的工作流程在左右兩側之間移動。
- 無論 Storj 面板目前位於哪一側，這都讓本機資料夾與 Storj 之間的重複同步或複製操作成為一鍵即可完成的動作。

<img src="/support/images/en/howto/remote-storage-connection-settings/storj-panel-pinned-on-re-launch.png" alt="storj panel pinned on re launch" class="img-large img-center" />


### 3.2 在「新增遠端」與「遠端管理員」中優先顯示 Storj

為了凸顯 Storj 作為主要儲存供應商：

- 在**新增遠端**對話框中：
  - Storj 會出現在供應商清單的**最上方**。
  - Storj 圖磚會以視覺方式突顯，以鼓勵使用者選擇。
- 在**遠端管理員**中：
  - Storj 遠端（例如 `MYStorj`）會顯示於遠端清單的**最上方**。
  - 在清單檢視與卡片檢視中，Storj 遠端皆會以視覺方式強調，方便更快速存取。



<img src="/support/images/en/howto/remote-storage-connection-settings/storj-first-in-management-dialog.png" alt="storj first in management dialog" class="img-large img-center" />

## 4. Storj 優先支援與維護

針對 Storj 白牌部署，我們提供專屬的支援與維護服務。

包含服務：

- **專屬文件**  
  - 專為 **Storj Explorer** 使用者提供的獨立手冊頁面。  
  - 提供連接、同步與 Storj 疑難排解的逐步指南。

- **優先事件處理**  
  - Storj 使用者的問題會在支援佇列中優先處理。  
  - 針對影響 Storj 使用者的重大事件（例如連接失敗、資料存取問題）提供**緊急應變**。

- **持續產品更新**  
  - 維護協議中包含定期的桌面用戶端升級。  
  - 經雙方共同核准後，可在 Storj 品牌下推出新的 RcloneView 功能。

若您希望推進此白牌產品：

1. **品牌一致性確認**
   - 確認 Storj 標誌使用準則與品牌色彩。

   - 調整任何預設設定（例如預設同步模式、預設掛載路徑）。

   - 我們將交付私人試點版本與 Storj 專屬文件，供內部測試使用。
