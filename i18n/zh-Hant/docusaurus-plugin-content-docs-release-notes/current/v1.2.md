# RcloneView 1.2 – 版本說明

**發布日期：** 2025 年 12 月<br/>
**平台：** Windows、macOS、Linux

RcloneView 1.2 針對工作執行、通知、雲端相容性與 UI 行為帶來重點改進。此版本著重提升長時間執行工作的可見性與控制能力，同時修正常見的工作流程問題。

:::info 通知與遠端控制改進
電子郵件與 Telegram 整合功能已擴充，提供更清楚的工作狀態通知。Telegram 現在不僅可用於通知，還能用來遠端控制工作執行。
:::

## 新功能

- **大型檔案傳輸的電子郵件通知：**
  現在可以針對大型檔案傳輸特別啟用電子郵件通知，有助於更有效地追蹤長時間執行或高影響性的工作。電子郵件轉送設定僅提供給 PLUS 使用者。

- **Telegram 通知與工作控制：**
  Telegram 整合功能已擴充，同時支援工作通知*與*遠端工作控制。您可以直接在 Telegram 上接收工作狀態更新，並啟動、停止或管理執行中的工作，而無需開啟應用程式介面。

- **新增 S3 供應商支援：**
  新增對另一個相容 S3 的儲存供應商的支援。

- **同步/複製工作的自訂旗標：**
  使用者現在可以在建立同步或複製工作時指定自訂的 rclone 旗標。

- **終端機選取內容複製：**
  內建終端機分頁中選取的文字現在可以複製到剪貼簿。

## 改進項目

- **電子郵件設定 UX 改進：** 改善電子郵件通知設定中的版面配置與用語，提升清晰度。
- **傳輸檢視中的工作名稱顯示：** 傳輸分頁現在會顯示進行中傳輸的工作名稱。
- **防止重複工作與排程器執行：** 防止同一工作多次執行，包括排程執行的情況。
- **遠端資料夾名稱可見性：** 修正「新增工作」與「工作管理員」檢視中的截斷問題。
- **試跑「查看詳情」欄位自動調整大小：** 自動展開最後一欄以避免文字被截斷。
- **新增遠端對話框對齊：** 改善圖示與文字的對齊方式，使版面更整潔。
- **即將推出功能面板優化：** 已被採用答案的主題現在會被隱藏。
- **macOS /Volumes 資料夾偵測：** 自動偵測 `/Volumes` 下的新增與移除項目。

## 錯誤修正

- **OneDrive 檔案刪除：** 修正無法從 OneDrive 遠端刪除檔案的問題。
- **Google Drive 權杖錯誤（外部 rclone）：** 解決使用無介面 Google Drive 權杖時發生的連線錯誤。
- **傳輸狀態不一致：** 修正傳輸已完成卻仍顯示「進行中」的情況。
- **工作歷史狀態修正：** 執行期間被強制終止的工作不再持續標示為執行中。
- **本機磁碟雜湊比對卡住：** 修正在本機磁碟上進行雜湊比對時可能卡住的問題。
- **「無變更」狀態一致性：** 無論整體工作是否完成，「無變更」狀態現在都能正確保留。
- **傳輸取消工具提示位置：** 修正取消按鈕的工具提示顯示位置。
- **重複遠端註冊：** 防止在以相同名稱重新新增遠端時重複使用過期的檔案清單。
- **rclone 設定相容性更新：** 更新設定值的處理方式，以配合 rclone 版本的變更。
