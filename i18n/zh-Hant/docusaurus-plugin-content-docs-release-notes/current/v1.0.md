# RcloneView 1.0 – 發行說明

**發行日期：** 2025 年 12 月<br/>
**平台：** Windows、macOS、Linux（包含 Raspberry Pi OS）

RcloneView 1.0 是一個重要里程碑版本，帶來更強的穩定性、更廣泛的平台支援、改良的遠端管理，以及更簡潔、資訊更豐富的介面。此版本根據使用者回饋與實際使用情況，加入了新功能、使用者體驗改善與重大錯誤修正。

## 新功能

- **支援複製工作類型：** 在工作管理員中新增「Copy」工作類型，讓遠端之間的單向檔案傳輸設定更加容易。
- **自動偵測網路磁碟機（Windows）：** RcloneView 現在會自動偵測並顯示已對應的網路磁碟機（例如 SMB 共用），無需使用者手動操作。這些磁碟機會直接顯示在**本機**分頁中，方便立即用於同步、比較與掛載。
- **右鍵選單新增「取得大小」：** 右鍵選單中新增「取得大小」選項，可快速計算所選資料夾的大小。
- **即將推出的功能面板：** 底部記錄視窗新增一個面板，顯示已規劃及即將推出的功能清單，協助使用者掌握未來更新資訊。

## 介面與使用者體驗改善

- **傳輸視窗重新設計：** 全面翻新傳輸工作介面，版面更加簡潔，最多可顯示 100 筆項目，提升效能與可讀性。
- **工作狀態顯示優化：** 改良工作狀態指示器，讓進行中與已完成任務的監控更準確、更直覺。
- **檔案總管樹狀結構行為：** 樹狀檢視在初次啟動時會自動展開根資料夾，方便瀏覽。同時修正了版面窗格中資料夾名稱重疊的問題。
- **掛載介面改善（Windows/macOS）：** 修正在 Windows 上選取本機資料夾時的版面問題，並新增掛載/取消掛載時本機磁碟分頁自動重新整理功能。同時修正掛載路徑更新行為，以及 macOS 上多重遠端 `cmount` 的問題。
- **遠端選取與顯示修正：** 改良遠端選取對話框，在同步與比較操作期間可正確顯示雲端硬碟。

## 錯誤修正

- **Synology NAS 自動偵測修正：** 解決偵測錯誤問題，並確保新增 NAS 遠端時能正確儲存預設位址。
- **重新啟動後預設遠端被重新指派：** 修正在使用額外視窗後重新啟動應用程式，會導致預設遠端連線被重設的問題。
- **工作分頁顯示問題：** 修正工作分頁中檔案與速度資訊遺失的問題。
- **AWS S3 行為修正：** 預設加入 `--no-check-bucket`，以避免同步錯誤。
- **Raspberry Pi OS 12 相容性：** RcloneView 現在可在 Raspberry Pi OS 12 上正常啟動並執行。
