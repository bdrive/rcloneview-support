# RcloneView 1.3 – 版本說明

**發布日期：** 2026 年 2 月<br/>
**支援平台：** Windows、macOS、Linux

RcloneView 1.3 新增了工作管理功能，並擴展了與 Slack 及 Discord 等熱門通訊工具的整合。此外，本版本也對電子郵件通知、UI 行為與工作執行追蹤進行了改進。

## 新功能

- **批次工作支援（Beta）：**
  現在您可以將多個工作分組為一個批次，並一起執行。
  這項 Beta 功能讓您能夠定義一系列依序執行的工作，更方便管理相關任務。
  未來的更新中將提供更多控制項與錯誤處理機制。

- **新增「移動」工作類型：**
  新增「移動」工作類型，讓您可以透過將檔案從來源移動到目的地來傳輸檔案。

- **重試失敗的工作：**
  現在您可以重試失敗的工作，而不需要手動重新建立。

- **Slack 整合：**
  RcloneView 現在支援透過 Slack 頻道接收通知並控制工作。

- **Discord 整合：**
  現在您可以透過 Discord 接收工作通知。

- **遠端控制設定指南位置調整：**
  Telegram 與 Slack 遠端控制設定指南的連結現在位於整合區段的頂部，更容易找到。

- **支援印尼語：**
  介面現已支援印尼語（Bahasa Indonesia）。

## 改進項目

- **電子郵件通知增強：**
  - 改善電子郵件設定面板的版面配置與清晰度
  - 工作狀態電子郵件現在包含詳細資訊
  - 修正電子郵件設定畫面中偶發的 UI 當機問題
  - 免費試用期結束後，電子郵件通知仍可繼續使用

- **遠端圖示更新：** 新增了新圖示並更新了舊圖示，方便識別遠端。

- **檔案清單載入穩定性：** 避免在檔案載入緩慢時切換頁面而顯示過期內容的情況。

- **「無變更」結果處理：** 確保無論工作完成狀態為何，「無變更」狀態都能在詳細資訊檢視中正確保留。

## 錯誤修正

- **OneDrive 檔案刪除：** 解決了在 OneDrive 遠端上無法刪除檔案的問題。
- **傳輸狀態顯示錯誤：** 修正了工作完成後檔案仍顯示為傳輸中的問題。
- **在本機磁碟上使用雜湊比對：** 修正了在本機儲存空間使用雜湊檢查比對檔案時發生的卡死問題。
- **工作歷程記錄不準確：** 強制終止的工作現在會正確地從歷程記錄中的「執行中」狀態移除。
- **傳輸取消提示位置錯位：** 修正了傳輸檢視中提示框的位置問題。
- **重新註冊同名遠端：** 避免以先前相同名稱新增遠端時重複使用過期的檔案清單。
- **Dry Run 欄位截斷：** Dry Run 詳細資訊面板中的最後一欄現在會自動展開以符合其內容。
- **外部 rclone 與 Google Drive 權杖錯誤：** 修正了在使用外部 rclone 搭配無頭模式（headless-mode）Google Drive 權杖時發生的連線問題。
